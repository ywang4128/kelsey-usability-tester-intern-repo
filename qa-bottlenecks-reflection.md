# Handling QA Bottlenecks in Sprints
## Why do QA bottlenecks happen in Agile teams?
late handoffs from devs team
over-optimistic commitments, no buffers
insufficient speint time for QA

## How can testers avoid testing everything at the last minute?
test small chunks throughout sprint
reserve 20% capacity for unplanned
daily standups flag ready stories

## What is parallel testing, and how does it help keep QA on track?
Run multiple tests concurrently to have faster feedback

## How do developers and testers collaborate to ensure smooth testing progress?
pair programming and testing sessions
shared test automation ownership
continuous integration + daily feedback
joint sprint planning and review

## What techniques can help QA work more efficiently without compromising quality?
risk-based prioritization
automation + AI test generation
clean test data

## If multiple features are ready for testing at the same time, how would you prioritize them?
I would prioritize them by risk(critical paths first), business impact, dependencied.

## How can early involvement of QA prevent last-minute testing rushes?
review stories day 1, prevent surprises, enables parallel work

## What would you suggest to a team if QA keeps blocking releases due to late testing?
shift-left testing, automation, buffers, cross-train team
