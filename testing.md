# Testing as an Investigative Process #26

## How does exploratory testing help testers uncover bugs that scripted tests might miss?
Exploratory testing lets tester interact with the app freely and creatively, trying unusual paths a real user might follow. It can help testers discover crashes, usability problems, and edge cases that automated or documented tests don't cover.

## What techniques can QA testers use to investigate software behavior systematically?
they can use techniques like session-based testing (setting time-boxed session), test charters(setting short mission), and heuristics(use simple rules of thumb) to keep testing focused but flexible, not just randomly clicking around.

## How do questioning skills help uncover hidden issues?
good questions help testers think like a user and a skeptic at the smae time.
Take Focus Bear for example, "what if the user does this instead of that?", "what happens if the network drops right here?", "is the behavior obvious enough for someone with ADHD or autism?"

## What strategies do expert testers use to test when there are no clear requirements?
concrete-driven testing
tester can look at the app's existing behavior and patterns to guess how it should work, talk to the product owner or designer to understand the goal, or test with concrete user-like scenarios.

## What questions would you ask when testing a new Focus Bear feature for the first time?
- what is the main goal of this screen for the user?
- how will this behave when the device is offline or the internet is slow?
- what happens if the user presses this button many times quickly?
- how clear is the wording for someone with ADHD or autism?

## How can uncertainty and curiosity make you a better tester?
Uncertainty and curiosity push me to dig deeper instead of assuming the app "works" because it passes the scripted tests.

## What techniques could you use to dig deeper when a bug is difficult to reproduce?
when a bug is flaky, I could try changing one factor at a time, writing down every step and data I used so I can repeat or narrow it down, or recording a short screen video and gathering logs so the developer can see exactly what happened.
